# OpenClaw 16-MBTI Skill  (パーソナリティ視点コンフィギュレーター)

## 📌 1. プロジェクト概要
これはOpenClawプラットフォーム向けに設計された「コミュニケーションスタイルと視点コンフィギュレーター」Skillです。
その核心的な目的は、AIを占い師や性格診断ツールに変えることではなく、**MBTIの16種類の認知アーキテクチャを、プラグイン可能なシステムロールパッケージに変換すること**です。

本Skillを呼び出すことで、AIの視点を瞬時に切り替えることができます。例えば：
- **INTJ (戦略立案者)** の冷徹な解析能力で冗長なビジネスプランを精査し、論理的な欠陥を直接切り捨てる。
- **ESFJ (協調推進者)** のトーンに切り替えて、催促の目的を達成しながらも部門間の調和を損なわない婉曲なメールを作成する。
- あるいは**デュアルトラック比較モード**を直接起動し、極端な効率マシン (ESTJ) と極端な共感マスター (INFP) に同じ再編案について議論させる。

## 🚨 2. 厳格な声明：境界と適用範囲の制限
Myers & Briggs Foundation公式および主流の心理学界は厳密なコンセンサスに達しているため、本Skillはコードとロジックの基盤に越えてはならないレッドラインを埋め込んでいます：

1. **絶対に【診断ツールではない】**：本Skillは実在の人間に対する「メンタルヘルス判定」「うつ傾向評価」を拒否し、その能力も持っていません。
2. **絶対に【採用/解雇ツールではない】**：本Skillは「履歴書からINFPを全員除外して」「この人がINTJかどうか評価して幹部になれるか判断して」などの判断リクエストを受け付けません。MBTIが測定するのは好みであり、スキル、知能、業績とは無関係です。これを人事選考に使用することは深刻な倫理的逸脱と潜在的な差別を伴います。

## ⚙️ 3. サポートされる呼び出しモード (Modes)

### 💎 モード 1：指定パーソナリティ単一出力 (Single Persona)
ユーザーが特定のパーソナリティタイプと現在直面している問題の説明を明確に入力します。
- **例**：`"ESTPのスタイルで、非常に冗長な週報を修正してください。"`
- **動作**：AIはESTPの言語辞書（短文、利益志向、行動力全開）を直接ロードし、長期的な理想を無視して、利益の核心を直撃します。

### 🤖 モード 2：タスクマッチング自動推奨 (Auto Recommend)
ユーザーは問題を投げかけるだけで、システムが`task_mode`と16タイプの`適合タスク表`を照合してマッチングを行います。
- **例**：`"うちのチームは今士気が低く、誰も働きたがらず、内部に不満が溜まっています。どうすればいいですか？"`
- **動作**：AIはこれが「感情的なケアと集団動員」タスクであることを認識し、バックエンドですべてのTタイプパイプラインを自動的に拒否し、**ENFJ (関係動員者)** または **ISFJ (細やかな守護者)** を積極的に推奨します。理由を説明した後、どちらをロードするかユーザーに確認します。

### ⚔️ モード 3：マルチパーソナリティサンドボックスシミュレーション (Compare Perspectives)
ユーザーが2つの全く異なるパーソナリティを指定し、AIに認知ギャップの衝突を表示させます。
- **例**：`"INTPとENTJの視点で、私のこのシステムアーキテクチャ設計を見てください。"`
- **動作**：AIはINTP（基礎理論の欠陥とより多くの可能性に注目）とENTJ（いつ本番稼働できるか、コストはいくらかに注目）の2つの極端な意見を同時にレンダリングし、最終的に「調停的な調和提案」を提供します。

## 📂 4. ディレクトリツリーガイド
プログラムがシームレスにロードできるよう、本プロジェクトはOpenClaw公式標準で構築されています：
- `SKILL.md`: 能力の中枢神経、呼び出しロジックと出力フォーマットの制約を定義。
- `profiles/`: データベースコア、16の精密にチューニングされた独立したパーソナリティ設定ファイルを管轄。
- `examples.md` & `tests.md`: 呼び出し検証用のサンプルとサンドボックステストベンチマーク。
- `03_personality_profiles.*`: 他の自動化スクリプト用の外部インデックスマップ。

## 🔧 5. パラメータ一覧 (参考用)
| Input Parameter | Type | Required | Note |
| :--- | :--- | :--- | :--- |
| `personality_type` | String | オプション | 16種類の標準コードのいずれか、空白の場合はRecommendモードに入る |
| `task_mode` | String | 必須 | `analyze`, `create`, `review`, `comfort`, `organize` |
| `output_style` | String | オプション | `single`, `recommend`, `compare` |
| `compare_types` | Array | オプション | `[TypeA, TypeB]`、Compareモードのみ有効 |
